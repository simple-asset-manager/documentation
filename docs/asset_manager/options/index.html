Options & Assumptions

The Options tab includes three important types of information:

  • Descriptive information such as the Property's address
  • Assumptions used for analysis and reporting such as dates, cap rate, and vacancy settings
  • Control over User Access & Permissions

Property#

Proforma Dates#

The Proforma Dates are the time period that is evaluated on the Proforma tab. The period selected will determine which Tenants and Expenses to include.

These fields include:

  • Start Date - The day the Proforma Period begins.
  • End Date - The day the Proforma Period ends.

Details#

The Property's descriptive information, such as the name and address.

The following fields are accepted:

  • Name
  • Type
    • Current options include Retail or Office
  • Address, City, State, Zip Code

Tenants#

Credit / Vacancy Loss#

Credit and Vacancy Loss is a reserve that is held to account for frictional vacancy, credit, and collection issues. This reserve is subtracted from Effective Gross Income to determine Potential Gross Income on the Proforma.

The following fields can be updated for each tenant:

  • Apply Vacancy - If set to On, a vacancy factor is applied to that tenant. If set to Off, no reserve is held for that tenant.
  • Vacancy Factor - The % of a tenant's rent payments assumed to not be collected due to vacancy, credit, and collection issues.

Valuation#

Properties are valued using the Capitalized Value approach. This approach is based on the theory that a property is worth the discounted sum of all its future payments.

This requires two inputs, a stabilized Net Operating Income and an assumed rate of return, more commonly known as the Cap Rate.

danger

The Capitalized Value approach assumes an unbroken revenue stream that grows at a constant rate.
It is crucial to:

  • Include only normal income and expenses
  • Set a proforma period that is one operating cycle, such as 12 months
  • Apply a reasonable discount rate

Cap Rate#

A Cap Rate is the assumed rate of return on an investment in a property. The Net Operating Income is divided by this rate to arrive at a valuation.

The following fields are accepted:

  • Cap Rate - The capitalization rate to apply to the property, between .01% and 100%. This should be entered as a percentage and not a decimal (7.50% is correct, 0.075 is not).
  • Comments - Any comments or support for the cap rate.
info

This approach is based on the Dividend Discount Model, which is used to value dividend paying stocks.

Click here if you'd like more information on the Dividend Discount Model (via Wikipedia).

Access & Permissions#

Access & Permission are two similar concepts, but have a very important distinction.

Access determines who can access the property.

Permissions determines what a user with access can do.

Who Can Access The Property?#

This section allows an Administrator to grant access, modify users' permissions, and revoke access.

Grant Access#

Access to the property can be granted to another user by sharing an Access Code. Each Access Code is tied to a specific permission, which determines what a user may or may not do - such as update tenant data, manage payments, bill tenants, send messages, and more. It is considered best practice to grant the minimum level of permissions a user requires to successfully perform their task.

Each permission, and its associated Access Code, is listed in this section. Click Permission Details to view a summary of a permission,

Access Codes are entered by clicking "Add Property" and then "Enter Property Code" from the main Portfolio page.

danger

There is one Administrator Access created when a property is first added. This user can perform any function related to the property, including granting, modifying or revoking user access. Be very careful when assigning Administrator Access to a user.

Edit Access#

Select any user from the dropdown to change their assigned permission or revoke access to the property entirely.

To change the user's property permissions, select the desired permission from "Select New User Permission". Once a new permission is selected, a comparison is provided that highlights how permissions will change from their previously assigned permission.

To revoke a user's access to the property - click "Revoke Access" - and that user will no longer be able to access or perform any actions related to the property.

View Access#

Select any user from the dropdown to view their assigned permission and what actions they can perform.

What Can They Do?#

Permissions determine what actions a user granted access to the property can perform. This includes billing, updating tenant data, uploading and sharing documents, messaging tenants, and more.

Create Permission#

Permissions can be fine tuned to suit any role in your organization.

To create a permission:

  1. Enter a name for the permission
  2. Select "Permit" or "Deny" for each relevant action. These include:
    • Tenant Management
      • Charges - Requesting, Scheduling, and Deleting Charges, entered in Tenant Payments under the Payments tab.
        • Create and Post Tenant Charges
        • Delete Tenant Charges
      • Data - All tenant information entered in the Tenants tab.
        • Create New Tenants
        • Delete Tenants
        • Edit Tenant Data
      • Documents - Documents stored and shared between Landlord and Tenant.
        • Upload and Share Documents
        • Delete Documents
      • Messages - Communication between the property and tenant in the Messages tab.
        • Send Messages
        • View Messages
    • Property Management
      • Expenses - Property Expenses entered in the Expenses tab.
        • Create Property Expenses
        • Delete Property Expenses
        • Edit Property Expenses
      • Payments - Property payment settings, such as which Bank Account to deposit rent payments, entered in Account Details under the Payments tab.
        • Link Banks Accounts & Make Payments
        • Unlink Bank Accounts
        • Edit Banking and Payment Information
      • Property - Creating, Deleting, and Updating Properties, via the main Portfolio page.
        • Create New Properties
        • Delete Properties
        • Edit Property Details
      • Proforma & Valuation - Updating the Proforma Date Period and updating Cap Rate information, via the Options tab.
        • Create Cap Rate
        • Delete Cap Rate
        • Update Proforma Dates
  3. Click "Save".

Edit Permission#

Permissions can be updated after they are created if a certain role needs to be expanded or changed.

To edit a permission:

  1. Select the permisison to edit from the dropbox box.
  2. Update either the permission name or what actions the permission permits or denies.
  3. Click "Save".

View Permission#

View any permission that has been created to make sure it covers everything the role needs.
